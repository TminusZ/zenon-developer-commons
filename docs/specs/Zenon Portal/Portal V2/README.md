# Zenon Portal Specification — Version Notice

A **V2 version of the Zenon Portal specification** has been published following a clarification issue discovered during review of Zenon’s original documentation.

Specifically, references made by Zenon core developers to **“satellites”** and the **“Satellite SDK”** may not have been intended in the literal sense of separate satellite infrastructure nodes. It is possible these terms instead describe **a network actor role within a data availability (DA) layer** of the Zenon network.

If this interpretation is correct, it has architectural implications for how the Portal protocol interacts with Zenon’s network and how certain infrastructure assumptions should be understood.

Because the original documentation leaves room for interpretation, **both versions of the specification are preserved**. The V2 revision adjusts the architectural assumptions to account for the possibility that “satellites” represent **a DA-layer network role rather than literal infrastructure nodes**.

Both versions are intentionally left available so that **the community, researchers, and implementers can determine whether the specification is usable and which interpretation—if any—accurately reflects Zenon’s intended architecture**.

This specification is independent research and should not be interpreted as official Zenon documentation.
